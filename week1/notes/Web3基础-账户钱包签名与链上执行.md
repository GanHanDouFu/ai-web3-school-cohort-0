# Module B：Web3 基础 - 账户、钱包、签名与链上执行

## 1. 钱包到底是什么

### 一句话理解

钱包 = 私钥管理器 + 签名工具。它不存钱，钱在链上。

### 类比

| 传统银行 | Web3 钱包 |
|----------|-----------|
| 银行帮你保管钱 | 钱包只保管钥匙 |
| 银行知道你的密码 | 没人知道你的私钥（除了你） |
| 银行能冻结你的账户 | 没人能冻结你的链上地址 |
| 忘了密码找银行重置 | 忘了私钥 = 永久丢失资产 |

### 密钥体系

```
助记词 (12个英文单词)
    "witch collapse practice feed shame open despair creek road again ice least"
    ↓ BIP-39 算法推导
私钥 (256位二进制，64位十六进制)
    "af4b3c5d...很长一串"
    ↓ 椭圆曲线算法 (secp256k1)
公钥
    "04ab2c3d...更长一串"
    ↓ Keccak-256 哈希，取后20字节
地址 (0x开头，42个字符)
    "0xF0cf20f5821fFe917F24532075E2FAC81b580c2e"
```

**关键理解**：
- **助记词** = 私钥的可读备份，12 个单词比 64 位十六进制好记
- **私钥** = 真正的控制权，谁有私钥谁就能操作资产
- **地址** = 公开身份，可以告诉别人，别人可以给你转账

### 为什么助记词不能泄露

泄露助记词 = 泄露私钥 = 失去所有资产控制权 = 不可找回

**真实案例**：有人截图助记词存在手机相册，手机被黑后资产被盗。有人把助记词发到群里"求助"，几秒内资产被转走。

## 2. 签名 = 授权一个具体动作

### 理解

签名不是"点确认"，而是用私钥对一段数据进行加密，证明"我授权这个动作"。

**类比**：签名就像在合同上签字。签字之前你得看清楚合同内容，签了就表示你同意。

### 签名的三步

```
1. 构造交易数据（转多少钱、给谁、Gas 费多少）
2. 用私钥对数据签名（MetaMask 弹窗让你确认）
3. 签名后的交易广播到网络（不可撤销）
```

### 签名前必须检查

- **To**：转给谁？是不是你想转的地址？
- **Value**：转多少钱？数量对不对？
- **Data**：如果是合约调用，data 字段是什么意思？
- **Gas**：手续费合理吗？

## 3. Gas = 链上计算的成本

### 为什么需要 Gas

链上每一步操作都要消耗计算资源。Gas 就是给这些资源定价。

**类比**：就像开车要耗油。路越远（操作越复杂），耗油越多（Gas 越高）。

### Gas 怎么算

```
总费用 = Gas Used × Gas Price

Gas Used：实际消耗的计算步骤（固定的，由操作复杂度决定）
Gas Price：每步的价格（浮动的，由网络拥堵程度决定）
```

**例子**：
- 转账：Gas Used = 21000（固定）
- 部署简单合约：Gas Used ≈ 200000
- 部署复杂合约：Gas Used 可能达到几百万

### L1 vs L2

| | L1 (以太坊主网) | L2 (Arbitrum 等) |
|--|-----------------|------------------|
| 安全性 | 最高 | 继承 L1 安全性 |
| Gas 费 | 贵（几美元到几十美元） | 便宜 10-100 倍 |
| 速度 | 12 秒一个区块 | 更快 |
| 用途 | 高价值交易、最终结算 | 日常交易、小额操作 |

### 我观察到的 Gas

- 测试交易 Gas：21000 × 8.45 Gwei = 0.000177 ETH
- 合约部署 Gas：199056 × 3.6 Gwei = 0.000717 ETH
- 合约部署比普通转账贵约 4 倍（因为要存储代码）

## 4. 智能合约 = 链上自动执行的程序

### 与后端 API 的区别

| | 后端 API | 智能合约 |
|--|----------|----------|
| 代码 | 私有，只有开发者能看到 | 公开，所有人都能查看 |
| 状态 | 私有数据库 | 公开可查 |
| 执行 | 服务器内部运行 | 所有节点共同执行 |
| 修改 | 随时可改 | 需权限或不可改 |
| 成本 | 服务器费用 | 每次调用消耗 Gas |
| 确定性 | 可以有随机性 | 必须确定性（相同输入相同输出） |

### 我部署的 SimpleStorage 合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 private storedValue;  // 存储一个数字
    address public owner;         // 合约所有者

    event ValueChanged(address indexed updater, uint256 newValue);

    constructor() {
        owner = msg.sender;  // 部署者成为 owner
    }

    function set(uint256 x) public {
        storedValue = x;
        emit ValueChanged(msg.sender, x);  // 触发事件
    }

    function get() public view returns (uint256) {
        return storedValue;
    }
}
```

## 5. 测试网 vs 主网

| | 测试网 | 主网 |
|--|--------|------|
| 代币价值 | 无价值（免费领） | 真钱 |
| 用途 | 学习、测试、实验 | 真实资产交互 |
| 数据 | 可能被重置 | 永久保留 |
| Gas | 免费测试币 | 真钱 |

**结论**：所有学习和开发先在测试网完成，确认无误后再考虑主网。

## 6. 区块浏览器 = 链上 Google

Etherscan 能做什么：
- 查看任何交易的详情（哈希、状态、Gas、区块）
- 查看任何地址的余额和交易历史
- 查看合约代码和状态
- 验证合约源码

**关键理解**：区块链是公开的，任何人都能查看任何交易。地址是假名，不是匿名。

## 7. 高级概念（了解即可）

### 账户抽象 (ERC-4337)

普通账户（EOA）只能由私钥控制。账户抽象让合约也能发起交易，支持：
- 社交恢复（朋友帮你找回账户）
- 批量交易（一次操作完成多步）
- Gas 代付（别人帮你付手续费）
- 为 AI Agent 提供可编程的权限控制

### 为什么对 AI × Web3 重要

AI Agent 需要有限权限（不能全权控制资产）。账户抽象 + 多签可以实现：
- AI 作为其中一个签名者
- 设置每日操作限额
- 关键操作需人工确认
- 操作记录可审计
